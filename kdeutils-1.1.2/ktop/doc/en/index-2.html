2. Installation

2.1 How to obtain KTop

KTop is part of the KDE Project. KTop can be found on the KTop Homepage or KDE FTP Server, the main ftp site of the KDE Project.

2.2 Requirements

In order to successfully compile KTop, you need the KDE 1.x and Qt 1.42. All required libraries as well as KTop itself can be found on the KDE FTP Server.

At the moment the following flavors of UNIX are supported:

  • Linux 2.x

    For KTop to work it is necessary to compile the Linux-Kernel with the /proc-File-system enabled. This is the default setting and most Linux-Distribution have it already. For SMP support Linux 2.2 is required.

  • FreeBSD

    The KTop program needs to be owned by the kmem group and needs to have the setgid bit set.

Support for other platforms is in preparation. Your help is greatly appreciated.

2.3 Compilation and installation

In order to compile and install KTop on your system, type the following in the base directory of the KTop distribution: 

% ./configure
% make
% make install

Should you run into problems please report them to the KDE mailing lists.

2.4 Installing the Task Manager Icon/Performance Meter in the KDE panel

It is possible to add the Task Manager to the panel. Simply use the Start Menu/Panel/Add Application Menu to do so. If you have the xload program installed you will see a performance history display instead of the KTop icon. Clicking on the icon or performance meter will start the Task Manager.

2.5 Reporting Bugs

No software is perfect and so KTop isn't. Although I made a lot of efforts to ensure the usability you might encounter a situation where KTop does not behave as you would expect. Please report all bugs to the highly sophisticated KDE Bug Tracking System. Do not report bugs directly to me. Reporting to the Bug Track System will ensure that all developers have a chance to read about it and fix it.

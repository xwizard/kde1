1. Introducción

ktalkd es un demonio de talk mejorado - el programa procesa las peticiones entrantes de talk, las anuncia y te permite responderlas con un cliente de talk. GLOSARIO : En estas páginas, si alguien quiere hablar contigo, se te designará como "llamado".

Ktalkd tiene las siguientes características :

KDE o no KDE

Esta es la cuestión. :) Más en serio, esto significa que Ktalkd puede ser compilado con o sin el soporte de KDE. Para usarlo sin KDE, el proceso de compilación detectará si las X, las Qt o las KDE no están instaladas, pero puedes forzar un demonio "no-KDE" si usas la opción de configuración '--without-X'.

Máquina contestadora

Si el llamado no esta conectado, o no responde tras el segundo anuncio, una "Máquina contestadora" arrancará, coje el mensaje y lo envía por correo al llamado.

Sonido

Si se quiere, puede sonar un sonido con el anuncio.

Anuncio en X

Si se compila con las KDE instaladas, Ktalkd usará ktalkdlg, un cuadro de KDE para el anuncio.

Anuncios en varios monitores

Si estás conectado de forma remota (ej. con un comando "export DISPLAY=..."), el aviso de X se hará en este monitor también. ¡ Responde donde quieras !

Reenvíando (Nuevo desde 0.8.0)

Puedes configurar un reenvío a otro usuario incluso a otra máquina si estás fuera. Hay tres métodos de reenvío diferentes. Mira en la sección "Uso".

Configuración

Si se compila con KDE, lee la información de los archivos de configuración de KDE, el entorno ($KDEDIR/share/config/ktalkdrc) y el del usuario, en su propio directorio principal.

El entorno uno debe de ser editado de forma manual por el administrador, pero ahora hay un cuadro de configuración para el usuario uno. Se llama kcmktalkd y puede encontrarse en el Centro de Control KDE después de instalar Ktalkd. En sistemas sin KDE, ktalkd leerá /etc/talkd.conf.

Internacionalización

Bajo KDE, el anuncio estará en tu idioma siempre y cuando así lo configures y alguien haya traducido el ktalkdlg a tu idioma. Lo mismo para el cuadro de configuración kcmktalkd.

Soporte para otalk y ntalk (Nuevo desde 0.8.1)

Ktalkd ahora soporta ambos protocolos, aun cuando se reenvíe. Ktalkd soporta también los dos.

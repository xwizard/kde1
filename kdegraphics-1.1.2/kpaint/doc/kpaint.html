KPaint - Help Contents
 

KPaint is a simple bitmap paint program. KPaint is currently beta quality software. There is of course no warranty as this is free software. For details of the license see the file LICENSE in the distribution. KPaint is part of the KDE Desktop Environment look on www.kde.org for more information.

KPaint requires version 1.2 of the Qt library from Troll Tech and libkde available from ftp.kde.org.

If you have any suggestions or bug reports my mail address is moorer@cs.man.ac.uk, please mention kpaint in the subject line of your message. More general dicussions about KDE should take place on the KDE mailing lists (link). If you are reporting bugs then please provide as much information as possible (including a stack trace if possible).

Currently there is no documentation but I'm starting to work on that, I've included an outline of how the help should look.

Differences from kpaint version 0.2

What's changed?
 

I've made further internal changes in this release to make kpaint much cleaner internally, though this process is not complete yet. You can save files properly (at last) setting the file types is no longer a problem, note though that I'm still using the Qt filedialog so the globs don't work right. I've also added network file access both load and save should work, but I still need to give visual feedback etc.

  • Load and Save now work properly (though there's no jpeg write support)
  • You can set the file types
  • Added a status bar
  • Fixed last window closed problem
  • Swanky new icon thanks to Nico Schirwing (nico.schirwing@bigfoot.com)
  • Network file access

Longer term plans are in the file PLANS in the source distribution
